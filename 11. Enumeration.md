## 정보수집
- 특정 대상의 정보를 수집해 추후 공격에 사용될 수 있는 정보를 파악
- `타겟 탐색` host IP, Network
- `타겟 분석` What Host?
- `타겟 공격 표면 확인` 어떤 port, 어떤 Netowrk service 실행 중?
- `타겟 공격 계획 수립` 수집된 정보를 바탕으로 어떻게 공격?

- 대상의 종류에 따른 정보 수집
  - 호스트: 포트스캐닝
  - 네트워크: Sweeping, DNS, 디렉토리 서비스 정보 수집
  - 웹: HTTP/S, websocket, gRPC 기반 정보 수집
  - 회사/기관: OSINT, 인적 자원, 기술 자원
- 목표 정하기: 공격 표면 생성 및 공격 계획 수립

### 기초 개념
- 서버: 다양한 기능들을 사용할 수 있도록 제공하는 컴퓨터
- 클라이언트: 사용자

- 포트: 서버에서 각각의 기능(프로세스)들을 분할하여 네트워크에 노출하기 위 특정 프로세스에 부여되는 논리적인 번호
  - `Web` 80
  - `DB` 3306
  - `Monitoring` 3000

  - `상태(status)`
    - `열림(Opened)` 들어오는 트래픽에 반응, 실행중인 네트워크 서비스 O
    - `닫힘(Closed)` 들어오는 트래픽에 반응X, 포트 닫혀 있음
    - `필터됨(Filtered)` 들어오는 트래픽에 반응할지 모름. 포트O, but 네트워크 서비스X

  - `프로토콜(Protocol)`
    - `TCP` TCP 3-way Handshake 과정을 이용해 포트 상태 확인
      - CONNENT
      - SYN
      - (Null, Fin, Xmas)등의 레거시
    - `UDP` 특정 페이로드를 사용해 각 포트 확인

### 포트스캐닝
- `$sudo wireshark` 모든 네트워크 트래픽을 확인할 수 있는 인터페이스(wireshark) 실행

- `nmap`
  - `$sudo namp -sn <ip주소>` 호스트 발견 스캔 `Host is up` ->  Host가 반응 O 
  - `$nmap <ip주소>` 가장 흔한 TCP포트 1000개를 뽑아 request 보내기 (connent scan)
  - `$sudo nmap -sS <ip주소>` (SYN scan, -sS)
  - `$sudo nmap -sU <ip주소>` (UDP scan, -sU)
 
  - `$nmap -p <포트1>,<포트2>,.. <ip주소>` <ip주소>의 <포트1>, <포트2>,..를 지정해서 스
