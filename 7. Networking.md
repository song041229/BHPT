## 네트워킹
- `ip 주소` 각각의 장치를 식별하고 위치를 지정하는데 사용되는 숫자 (고유한 주소)
  - `물리주소` 실제 주소
  - `가상주소` 부여된 가상 주소=ip 주소
  - `서브넷` ip 네트워크에서 더 작은 네트워크를 만드는 방법
 
- `게이트웨이` 서로 다른 네트워크 간 데이터 전달을 중개&프로토콜 변환&주소 변환하는 네트워크 주소
- `라우터` 게이트웨이의 하드웨어 장치

- `포트` 네트워크 통신에서 특정 프로세스나 서비스를 식별하기 위한 번호
  - `80` HTTP
  - `22` SSH
 
- `로컬호스트` 현재 사용중인 컴퓨터 or 장치를 가리키는 호스트(주소)
  - `127.0.0.1`라는 IPv4 주소로 매핑
  - `http` http://localhost
  - `ssh` user@localhost
 
- `DNS`
  - 네트워크에서 주소해석을 수행
  - 사용자가 이해하기 쉬운 도메인 이름 -> 컴퓨터가 이해할 수 있는 IP주소로 변환
 
- `방화벽`
  - 네트워크 트래픽을 제어하기 위해 사용되는 보안 장치
 
- 종합
  ```
  공격자 호스트 -> 라우터(게이트웨이) - 방화벽 타겟 호스트_포트
                         |
                      DNS 서버
  ```

## 네트워킹 명령어
- `$ipconfig`
- `$netstat` 네트워크&라우팅 테이블 정보
  - `Proto` 프로토콜 (tcp, udp)
- `$ping <target ip>` <target ip>와 연결 테스팅 용도
- `$tmux`
- `$nc` target <-> attacker로 파일 전송

- `etc/host` <ip>와 <host name>을 mapping
- `resolv.conf` host에서 실행하고 있는 DNS서버
